1.2 常見的NLP任務

  • 語言建模: 根據前面單字來預測句子接下來的單字，可以應用在語音辨識、光學字源辨識、機器翻譯、拼寫糾正。

  • 文本分類: 根據原文內容將原文分類至一組已知的類別，可應用在email垃圾信件辨識語情緒分析。

  • 資訊提取: 從原文裡頭提取相關資訊，例如找出email裡面的行事曆事件，或是找出社交媒體皆文中提到的人名。

  • 資訊檢索:從大集合中找出查詢指令相關的文件如google search

  • 對話代理人: 建構以人類語言交談的對話系統，如Siri

  • 原文摘要生成: 為較長的原文創造精簡的摘要並同時保留核心內容，以及整體的意思

  • 機器翻譯: 將原文轉成另一種語言，例如google translate這樣的工具就是使用這樣的技術

  • 主題建模: 找出同一群文件的主題結構，例如原文挖掘(text-mining)

2. 什麼是語言?

2.1 語言的成分

人類的語言分成四個主要成分:

  • 音素(phoneme): 又稱為音位，是人類語言中能夠區別意義的最小聲音單位，例如 bat’蝙蝠’ 和 cat’貓’兩個字中只有第一個子音是不一樣的，但是兩個字的意思卻截然不同，這樣一來我們就可以說b 跟 c 這兩個字的發音在英文中形成音素。

英文音素分布(節錄自維基百科)

  • 詞素(morpheme):最小的語義單位，由音素組成，並非所有詞素是單字，但是所有字首跟字尾都是詞素。例如cats中的s即是一個複數詞素，人們裡的們也是一個這樣的例子。一些相關的NLP任務包括:語一單元化(tokenization)、詞幹提取(stemming)、學習單字embedding、詞性標註。

  • 語法: 在語言學中指任意自然語言中句子、短語以及詞等文法單位的文法結構與文法意義的規律。語言學裡，語法結構有許多表達形式，一種常見的做法是用解析樹(parse tree)。不同的語言存在著不一樣的句法結構，如下圖:

英文語法結構

中文語法結構

  • 語境(context): 語境包含長期參考、世界知識、常識，以及單字語子句的字面意思。語境通常是由語意跟語用組成，語意指的是沒有上下文的情況下，單字與句子的直接意義。例如你可以開門嗎就字面上的意思，是在詢問一個人是否具備幫人開門的能力，但是如果有一個人大包小包然後他說出了你可以開門嗎，這時候他並不是在問你是否有能力開門，而是一個比較委婉的請求。

2.2 為何NLP具有挑戰性?

四大挑戰:

  • 語言模糊性: 人類的語言都是模稜兩可的，以書這個詞為例，當我們說有趣的書的時候，我們指的是書的內容，而我們說紅色的書指的卻是書的外殼。下圖是另一個有關於讀這個字的可能意思。

  • 常識: 常識是大部分人都知道的事實整合，但對於電腦來說具有極大挑戰性，舉例而言，當我們說，貓捉到老鼠因為牠跑得快，以及貓捉到老鼠因為牠跑得慢，這兩句的牠指的分別是貓以及老鼠，而之所以我們能知道這些差別，是因為我們具備常識，貓通常是捕食者所以速度一定要快，才能捉到被捕食者的老鼠。

  • 創造性: 語言並不是指根據規則，也會有一些創新說法，這些對於電腦而言也會是一個挑戰。例如吃爆、星爆等等。

  • 跨語言多樣性

    3. 機器學習、深度學習與NLP: 概要

    人工智慧(AI)是為建構一個系統來執行人類智慧的任務例如學習、問題解決、邏輯推理。機器學習(ML)是AI的分支，是希望透過在大量案例中學習，進而能自主解決任務的演算法，因而不需要人工規則。而深度學習(DL)則是機器學習的其中一個分支，採用人工神經網絡的結構。NLP早期是使用人工規則以及經驗法則，近十年開始使用ML以及DL的方法。

3.2 NLP的方法: NLP的機器學習

  • 單純貝式: 針對眼前發生的現象，運用過去相關的經驗（計算概度比），來理性地評估某事件可能發生的機率。例如在垃圾信中分類時，我們如果採用這樣的演算法，我們的問題可能就是: 如果一封信出現恭喜、難得、只有詞彙時牠是垃圾信件的機率有多少? (我們也必須知道恭喜、難得、只有這些詞出現在垃圾信件的機率有多高?)
  • 支援向量機: 可以學習線性以及非線性的邊界，並利用不同類別的點間的最大距離來找出最佳決策邊界。我們會需要找出一條線來區分我們的資料，如下圖。
  • 隱性馬可夫模型與隨機條件場: 這些都是統計模型，且大多都是運用在斷詞、指代消解上。

3.3 NLP的方法: NLP的深度學習

  • 摺積神經網路(CNN): 一開始是用於圖片或影像辨識，但後來也可以搭配使用在NLP，其主要優勢是在於能夠一次處理多字如下圖。
  • 遞迴神經網絡(RNN): 可用於處理具有順序性的資料，例如語言，它會一個字一個字看，並且短期記住上一個字，並更新資訊，該方法很適合運用在NLP相關任務，例如文本分類、機器翻譯等。運作機制如下圖。
  • 長短期記憶(LSTM): 遞迴神經網路的記憶大多都是短期，因此無法處理較長的原文，因此便有了LSTM，它能夠捨棄文中較不重要的部分，並牢牢記住真正需要的部分，運做方式如下圖。
  • transformer: 為近年較新的技術，當一個字進入時它會看附近所有的單字並且跟句上下字的意思來標示該單字，例如bank可能有河岸或銀行的意思，transformer就可以透過上下文的意思給這兩個意思不同標示。

    3.4 深度學習的限制

  • 當資料較少時容易產生overfit的問題
  • 小樣本學習: 視覺資料中透過少量訓練資料就可以達到高品質模型
  • 領域適應: 不同領域的資料推論效果不好
  • 可解釋的模型
  • 知識與常識
  • 成本: 硬體GPU採購，DL的部署與維護
